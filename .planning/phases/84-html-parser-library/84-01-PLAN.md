# Plan 84-01: HTML Parser Library

<objective>
Create pure TypeScript module `parseMonsterRuHtml(textEn, rusText) → MonsterStructuredLoc | null` with complete type definitions, barrel export, and a debug script that validates the parser deeply against the single Succubus fixture in `monster.json`. **Scope philosophy:** perfect the parser on Succubus (rich representative — prepared + innate spellcasting, multi-ability narrative, weaknesses, strikes) before scaling to more fixtures. Additional monsters deferred until `monster.json` is expanded in a follow-up phase/milestone.

The parser extracts structured RU localization data from a pair of EN/RU HTML stat blocks in markdown-lite format (see CONTEXT.md D-07). Runtime consumer code (Phase 86-88) reads the output directly without further parsing.

**Locked decisions (see 84-CONTEXT.md):**
- D-01..04: Minimum-viable type shape — `abilitiesLoc`, `skillsLoc`, `speedsLoc`, `savesLoc`, `acLoc`, `hpLoc`, `weaknessesLoc`, `resistancesLoc`, `immunitiesLoc`, `perceptionLoc`, `languagesLoc`, `strikesLoc`, `spellcastingLoc`. Numeric bonuses/damage NOT parsed.
- D-05: Name-match first, index fallback for abilities/skills/speeds
- D-06: Debug script (not Vitest), runs via `pnpm tsx`, zero new deps
- D-07..08: Markdown-lite output (`<b>X</b>` → `**X**`, `<i>X</i>` → `*X*`, `<br>` → `\n`, action markers extracted to `actionCount`)
</objective>

<threat_model>
**Input untrusted?** `rus_text` is bundled JSON (build-time included, not user input). Threat surface low — but parser still hardens:
- DOMParser isolates HTML into a detached `<body>`, no script execution (browser behavior)
- Output is structured data consumed by React via markdown-lite renderer (Phase 88) — renderer whitelist approach prevents injection
- No `dangerouslySetInnerHTML` in parser or its output consumers
- Pathological HTML (deeply nested, malformed) → graceful `null` return
</threat_model>

<action>
Create `src/shared/i18n/pf2e-content/lib/parse-monster.ts` with:

**Public API:**
```ts
export function parseMonsterRuHtml(
  textEn: string | undefined | null,
  rusText: string | undefined | null,
): MonsterStructuredLoc | null
```

**Implementation requirements:**

1. **Early return:** If `rusText` is falsy or `DOMParser.parseFromString(rusText, 'text/html')` throws → return `null`.

2. **Helper: `htmlToMarkdownLite(html: string): string`**
   - `<b>X</b>` / `<strong>X</strong>` → `**X**`
   - `<i>X</i>` / `<em>X</em>` → `*X*`
   - `<br>` / `<br/>` / `<br />` → `\n`
   - `<hr>` / `<hr/>` / `<hr />` → `\n---\n` (section marker — consumer can strip)
   - Extract `[one-action]` / `[two-actions]` / `[three-actions]` marker (count 1/2/3) — return tuple `{text: string, actionCount: 1|2|3|null}` when called on ability blocks
   - Unknown tags: strip tag, preserve inner text (`element.textContent` as fallback)
   - Collapse whitespace: `text.replace(/\s+/g, ' ').trim()` — but preserve `\n` from `<br>` conversion (use placeholder then restore)
   - Normalize entities: `&nbsp;` → ` `, `&amp;` → `&`, etc.

3. **Section extraction — operate on `document.body.childNodes`:**
   - Walk top-level nodes, split into logical sections by `<hr>` markers.
   - Typical pf2.ru layout (see Succubus in monster.json):
     - **Section 1 (before first `<hr>`):** Perception, Languages, Skills, Ability Scores — extract by bolded-key prefix scan
     - **Section 2 (between 1st and 2nd `<hr>`):** AC, Saves, HP, Weaknesses/Resistances/Immunities + pre-combat abilities (e.g. Seductive Presence, Rejection Vulnerability)
     - **Section 3 (after 2nd `<hr>`):** Speed, Strikes, Spellcasting, combat abilities (Change Shape, Embrace, Passionate Kiss, Profane Gift)
   - In-box abilities = `<span class="in-box-ability">` elements — extract separately throughout.

4. **Field-specific extractors:**
   - **`extractAbilities(rusBody, enBody)`:**
     - Collect `<span class="in-box-ability">` from both EN and RU bodies, in DOM order
     - For each RU ability: extract first `<b>Title</b>` → ability name; strip action markers → `actionCount`; parse traits from parenthesized list after title (e.g. `(аура, эмоции, ментальный)` → `["аура", "эмоции", "ментальный"]`); remainder → markdown-lite description
     - **Matching (D-05):**
       - Build lookup: `enAbilities` keyed by `normalizedName(ability.name)` (lowercase, collapse spaces, strip punctuation)
       - For i-th RU ability:
         - Try RU name → normalized key → if RU→EN dictionary has canonical mapping, use that EN ability as anchor
         - Else use i-th EN ability as positional fallback
         - If EN[i] doesn't exist (RU has more) → still include in output; Phase 88 uses RU-only overlay
     - Return `AbilityLoc[]` preserving RU order
   - **`extractSkills(rusBody)`:**
     - Find `<b>Навыки</b>` (or case-insensitive variants) → parse comma-separated list → regex `(\S[^+]+?)\s*([+-]?\d+)` → `{name: "Акробатика", bonus: 14, engineKey: skillRuToEn[name]}`
     - Skill dictionary (RU→EN) — hardcode 17 pf2e skills:
       ```
       Акробатика→Acrobatics, Искусство→Arcana, Атлетика→Athletics,
       Ремесло→Crafting, Обман→Deception, Дипломатия→Diplomacy,
       Запугивание→Intimidation, Медицина→Medicine, Природа→Nature,
       Оккультизм→Occultism, Исполнение→Performance, Религия→Religion,
       Общество→Society, Скрытность→Stealth, Выживание→Survival,
       Воровство→Thievery, Внимание→Perception
       ```
   - **`extractSpeeds(rusBody)`:**
     - Find `<b>Скорость</b>` section
     - Parse comma-separated tokens; key phrases: bare number = `land`, `полёт N` = `fly`, `лазание N` = `climb`, `рытьё N` = `burrow`, `плавание N` = `swim`
     - Return `SpeedsLoc` with string values ("25 футов")
   - **`extractSaves(rusBody)`:** Find `<b>Стойкость</b>`, `<b>Реакция>`, `<b>Воля</b>` labels; return `{fortLabel, refLabel, willLabel}` as found. If absent, use defaults: `"Стойкость"`, `"Реакция"`, `"Воля"`.
   - **`extractAC(rusBody)`:** Find `<b>КБ</b>` label. Return `{label: "КБ"}` (or found variant).
   - **`extractHP(rusBody)`:** Find `<b>ПЗ</b>` label. Return `{label: "ПЗ"}` (or found variant).
   - **`extractWeaknesses/Resistances/Immunities(rusBody)`:**
     - Find `<b>Уязвимости</b>` / `<b>Сопротивления</b>` / `<b>Иммунитеты</b>` followed by comma list
     - Return `string[]` of raw tokens (e.g. `["холодное железо 5", "священный 5"]`)
   - **`extractPerception(rusBody)`:** Find `<b>Внимание</b>` section → `{label: "Внимание", senses: "ночное зрение"}`
   - **`extractLanguages(rusBody)`:** Find `<b>Языки</b>` → comma-split → `string[]`
   - **`extractStrikes(rusBody)`:**
     - Find `<span class="in-box-ability">` containing `<b>В ближнем бою</b>` or `<b>На расстоянии</b>`
     - Extract strike name (word after action icon, before `+` or bonus) → `name`
     - Extract damage type suffix (word at end of "Урон Nd+M тип") → `damageType`
     - Return `StrikeLoc[]`
   - **`extractSpellcastingHeading(rusBody)`:**
     - Find first `<b>...заклинания</b>` pattern (e.g. "Врождённые сакральные заклинания", "Подготовленные сакральные заклинания", "Спонтанные первородные заклинания")
     - Return `{headingLabel: <found text>}`, or default `{headingLabel: "Заклинания"}` if none.

5. **Partial parse resilience:** Each extractor wraps its body in `try { ... } catch { return <field default> }`. Field default examples: `abilitiesLoc: []`, `skillsLoc: []`, `weaknessesLoc: []`, `savesLoc: { fortLabel: "Стойкость", refLabel: "Реакция", willLabel: "Воля" }`, `spellcastingLoc: { headingLabel: "" }`. Only a full-DOMParser-throw returns root `null`.

6. **No React, no Tauri APIs, no network.** Pure browser-TS using only DOMParser + standard string/regex ops.

7. **Character count target:** Main `parse-monster.ts` ≤ 450 lines incl. JSDoc and dictionaries. If approaching, extract helpers into `parse-monster-helpers.ts` — acceptable; flag to reviewer.

Add JSDoc block at top of file explaining:
- Purpose (summary)
- Contract (input = HTML string pair, output = structured or null)
- When `null` returned vs when empty arrays
- Why numeric values not parsed (D-03 rationale)
