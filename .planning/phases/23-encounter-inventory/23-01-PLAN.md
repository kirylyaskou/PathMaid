# Plan 23-01: Per-Encounter Inventory Overrides

**Phase:** 23 — Encounter Inventory Editor
**Requirements:** ENCINV-01, ENCINV-02, ENCINV-03
**Status:** Not started

## Goal

DM can add/remove items from a creature's inventory for a specific encounter without modifying the base creature. Same non-destructive override pattern as spell overrides (Phase 19).

## Changes

- `src/shared/db/migrations/0012_encounter_item_overrides.sql`:
  - `encounter_combatant_items` table:
    - encounter_id TEXT, combatant_id TEXT, item_name TEXT, item_foundry_id TEXT (nullable), item_type TEXT, quantity INTEGER DEFAULT 1, damage_formula TEXT, ac_bonus INTEGER, is_removed INTEGER DEFAULT 0
    - PK: (encounter_id, combatant_id, item_foundry_id) — or UUID for added items with no foundry_id
    - `is_removed=1` → hide a base item; `is_removed=0` → added item for this encounter only

- `src/shared/api/encounters.ts`:
  - `EncounterItemRow` interface
  - `loadItemOverrides(encounterId, combatantId)` — SELECT from encounter_combatant_items
  - `upsertItemOverride(encounterId, combatantId, item)` — INSERT OR REPLACE
  - `deleteItemOverride(encounterId, combatantId, itemFoundryId)` — DELETE by PK
  - `resetEncounterCombat` extended to clear `encounter_combatant_items WHERE encounter_id=?`

- `src/entities/creature/ui/CreatureStatBlock.tsx`:
  - `EquipmentBlock` extended with `encounterContext` prop (same interface as SpellcastingBlock)
  - When encounterContext provided: loads overrides on mount, merges with base inventory
    - Base items with `is_removed=true` override hidden (× button applied)
    - Added items (is_removed=false) appended to list
  - "Add Item" row at bottom (when encounterContext): text search input using `searchItems`, select from dropdown, appends to override list
  - Hover-reveals × button on each item to mark as removed (base items) or delete (added items)

## Notes

- Override state managed with useState + useEffect pattern (same as SlotPips in SpellcastingBlock)
- Optimistic UI: update local state immediately, then persist async
- "Add Item" search: `searchItems(query)` from shared/api, shown as autocomplete list (max 8 results)
- UUID for added items without foundry_id: `crypto.randomUUID()` at insert time
