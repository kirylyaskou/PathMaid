# Plan 27-01: Condition Badge → Detail Popover

**Phase:** 27 — Condition Badge Integration
**Requirements:** CONDB-01, CONDB-02
**Status:** Complete

## Goal

Clicking a condition badge in the combat tracker opens a popover showing the condition's description excerpt, modifier summary, and overrides — without leaving the combat view. Data comes from the `conditions` table populated in Phase 25.

## Changes

- `src/entities/condition/model/types.ts`:
  - Add `import type { ConditionRow } from '@/shared/api'`
  - Export `ConditionRow` type from barrel

- `src/entities/condition/ui/ConditionBadge.tsx`:
  - Add optional `onInfo?: () => void` prop
  - When `onInfo` provided, render a small info icon button (ⓘ) alongside the badge
  - Does NOT make the whole badge clickable (keeps existing remove-on-click behavior intact)

- `src/widgets/combatant-detail/ui/ConditionSection.tsx`:
  - Import `getConditionBySlug` from shared/api
  - Add `selectedSlug: string | null` + `conditionDetail: ConditionRow | null` state
  - When info icon clicked on a badge: fetch `getConditionBySlug(slug)` → store in conditionDetail
  - Render `ConditionDetailPopover` component when conditionDetail is set

- New local component `ConditionDetailPopover` (inline in ConditionSection.tsx):
  - Fixed panel below condition list (not floating — avoids z-index issues in panel layout)
  - Header: condition name + valued badge + close button (×)
  - Modifier summary line (if any): "Status −value to all checks"
  - Overrides list (if any): "Overrides: Dazzled"
  - Description: first 300 chars from stripHtml(resolveFoundryTokens(description)) + "…"
  - Link: "See Conditions page" (navigate to /conditions)

## Notes

- ConditionDetailPopover is NOT a tooltip — it's a persistent panel that stays open until × clicked or another badge info is clicked
- Only one condition detail open at a time (selectedSlug replaces previous)
- Fallback: if no DB row found for slug (e.g., persistent-fire not in conditions table), show name + "No reference data available"
- resolveFoundryTokensForDisplay already exists in CreatureStatBlock — copy same local impl into ConditionSection.tsx
- Description excerpt: 300 chars after stripHtml → preserve readability
