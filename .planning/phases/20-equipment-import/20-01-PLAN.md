# Plan 20-01: DB Migration + Equipment Extraction

**Phase:** 20 — Equipment Import Pipeline
**Requirements:** EQUIP-01, EQUIP-02, EQUIP-03
**Status:** Complete

## Goal

SQLite schema for items + FTS5, plus extraction of all equipment entity types from Foundry VTT raw_json during sync. Mirrors the spells pipeline (0009_spells.sql + extractAndInsertSpells).

## Changes

- `src/shared/db/migrations/0011_items.sql`:
  - `items` table — id, name, item_type, level, rarity, bulk, price_gp, traits, description, source_book, source_pack
  - Type-specific columns on same table: damage_formula, damage_type, weapon_category, weapon_group, ac_bonus, dex_cap, check_penalty, speed_penalty, strength_req, consumable_category, uses_max
  - `items_fts` FTS5 virtual table (name, traits, content=items)
  - `creature_items` table — id, creature_id, item_name, item_type, foundry_item_id, quantity, bulk, damage_formula, ac_bonus, traits, sort_order

- `src/shared/api/sync.ts`:
  - `ITEM_TYPES` const: `['weapon','armor','consumable','equipment','treasure','backpack','kit','book','shield']`
  - `parseItemPrice(price)` helper — converts `{gp:N, sp:N, cp:N, pp:N}` to total gp float
  - `extractAndInsertItems(entities)` — filters entity_type in ITEM_TYPES, parses system.* fields, batch-inserts into `items`, rebuilds FTS5
  - `extractCreatureItems(entities)` — filters entity_type='npc', reads items[] from raw_json, extracts non-strike/non-spell items (weapon, armor, equipment, consumable, backpack, treasure), batch-inserts into `creature_items`
  - Both called at end of `syncFoundryData` and `importLocalPacks`

## Notes

- `bulk` stored as TEXT ("L", "1", "2", "-") — not parsed to float
- `damage_formula` = "{dice}d{die} {damageType}" e.g. "1d8 slashing"
- `price_gp` rounded to 2 decimal places (cp/10, sp/100, pp*10 conversions)
- `description` stored as raw HTML — cleaned at display time by stripHtml
- Creature items skip item_type='melee' and 'ranged' (already handled as strikes) and 'spellcastingEntry'/'spell'
- `foundry_item_id` extracted from item `flags.core.sourceId` or `_id` field
