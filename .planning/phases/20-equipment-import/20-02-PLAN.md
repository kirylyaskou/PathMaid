# Plan 20-02: shared/api/items.ts

**Phase:** 20 — Equipment Import Pipeline
**Requirements:** EQUIP-01, EQUIP-02, EQUIP-03
**Status:** Complete

## Goal

Frontend API layer for items — search, filter, single fetch, count. Mirrors spells.ts pattern exactly.

## Changes

- `src/shared/api/items.ts` (new):
  - `ItemRow` interface — mirrors `items` table columns
  - `CreatureItemRow` interface — mirrors `creature_items` table columns
  - `searchItems(query, itemType?, minLevel?, maxLevel?, rarity?)` — FTS5 search with optional filters, LIMIT 100; falls back to `SELECT * FROM items WHERE ...` if query empty
  - `getItemById(id)` — single item by PK
  - `getItemsByType(itemType)` — all items of a given type
  - `getItemCount()` — COUNT(*) from items
  - `getCreatureItems(creatureId)` — returns `CreatureItemRow[]` ordered by sort_order

- `src/shared/api/index.ts` — add `export * from './items'`

## Notes

- FTS5 query: `SELECT i.* FROM items_fts fts JOIN items i ON i.rowid = fts.rowid WHERE fts MATCH ? [AND item_type=?] [AND level BETWEEN ? AND ?] ORDER BY rank LIMIT 100`
- Empty query uses `SELECT * FROM items WHERE 1=1 [AND item_type=?] [AND level BETWEEN ? AND ?] [AND rarity=?] LIMIT 100`
- Same getSqlite() raw SQL pattern as spells.ts
