# Plan 26-01: ConditionsPage — Full Reference UI

**Phase:** 26 — Conditions Reference Page
**Requirements:** CONDP-01, CONDP-02, CONDP-03
**Status:** Not started

## Goal

Replace ConditionsPage stub with a full conditions reference catalog — searchable, grouped, with expandable mechanical detail cards.

## Changes

- `src/pages/conditions/ui/ConditionsPage.tsx` — replace placeholder with:
  - Search input (simple filter on name, no debounce needed for ~50 items)
  - Group filter tabs: All / Death / Abilities / Senses / Detection / Attitudes / Other
  - Sorted list of ConditionCard rows (alphabetical within group)
  - Footer: "N conditions" count
  - Empty state: "Run sync to import conditions"
  - Loads via `getAllConditions()` once on mount

- Local `ConditionCard` component (inline in page):
  - Collapsed row: name, valued badge ("1–4", or "✓" for non-valued), group chip, modifier_summary (truncated)
  - Expanded section:
    - Full HTML description rendered via `stripHtml` + `resolveFoundryTokensForDisplay`
    - Overrides list: "Overrides: Dazzled, ..." (if any)
    - Modifier summary block (formatted): each modifier on its own line
    - Source book (small, muted)
  - Valued badge colors: orange for valued, green for non-valued
  - Group chip colors: death=red, abilities=purple, senses=blue, detection=cyan, attitudes=amber, other=zinc

## Notes

- Load all conditions once on mount (getAllConditions) — no pagination needed
- Client-side filter on name + group (no DB round-trips on filter change)
- resolveFoundryTokensForDisplay already exists in CreatureStatBlock — copy same inline impl into ConditionsPage
- Description shown as plain text (stripHtml applied), not raw HTML
- Overrides parsed from JSON: `JSON.parse(condition.overrides ?? '[]')`
