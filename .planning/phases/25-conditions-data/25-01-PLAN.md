# Plan 25-01: Conditions DB Migration + Extraction

**Phase:** 25 — Conditions Data Pipeline
**Requirements:** COND-01, COND-02
**Status:** Not started

## Goal

Dedicated `conditions` table parsed from Foundry VTT condition entities — with structured mechanical data (rules_json, modifier_summary, is_valued, overrides) ready for both the reference page and future modifier engine integration.

## Changes

- `src/shared/db/migrations/0013_conditions.sql`:
  - `conditions` table:
    - id TEXT PK (Foundry _id)
    - name TEXT
    - slug TEXT (system.slug or lowercase name)
    - is_valued INTEGER (0/1 — system.value.isValued)
    - description TEXT (system.description.value HTML)
    - group_name TEXT (system.group: "senses", "abilities", "death", "detection", "attitudes", null)
    - overrides TEXT (JSON array of slugs this condition overrides, e.g. ["dazzled"])
    - rules_json TEXT (system.rules[] full JSON — for future engine modifier integration)
    - modifier_summary TEXT (pre-computed human-readable modifier string)
    - source_book TEXT (system.publication.title)
  - INDEX on slug, group_name

- `src/shared/api/sync.ts`:
  - `parseModifierSummary(rules)` helper — converts rules[] FlatModifier entries to readable string:
    - selector "all" → "all checks"
    - selector "perception" → "Perception"
    - selector "attack-roll" → "attack rolls"
    - selector array → join with " and "
    - value "-@item.badge.value" → "−value" (dynamic)
    - value N → "−|N|" (static)
    - type "status" → "Status"
    - type "circumstance" → "Circumstance"
    - Multiple modifiers joined with "; "
  - `extractAndInsertConditions(entities)` — filters entity_type='condition', parses system.* fields, batch-inserts into conditions table
  - Called at end of syncFoundryData + importLocalPacks (after extractAndInsertItems)

## Notes

- Conditions are already in entities table — this is additive extraction (no deletion from entities)
- slug: prefer system.slug if present, else name.toLowerCase().replace(/ /g, '-')
- rules_json stores FULL system.rules[] — not just FlatModifier — so future integration can use GrantItem, Immunity, etc.
- modifier_summary = '' if no FlatModifier rules (non-valued binary conditions like Prone: still has modifiers though)
- Re-extract every sync (DELETE FROM conditions before re-insert)
