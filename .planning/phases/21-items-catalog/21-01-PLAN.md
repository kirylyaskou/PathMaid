# Plan 21-01: ItemCard Component + Filter Model

**Phase:** 21 — Items Catalog Page
**Requirements:** ITMCAT-01, ITMCAT-02, ITMCAT-03
**Status:** Not started

## Goal

Reusable ItemCard component (expandable detail row) and filter state model for the items catalog. Foundation for the full page in 21-02.

## Changes

- `src/entities/item/model/types.ts` — replace stub with full `ItemRow` import re-export from shared/api/items; add `ITEM_TYPE_LABELS` map for display names; add `ItemTypeFilter` type

- `src/entities/item/ui/ItemCard.tsx` (new):
  - Props: `item: ItemRow, expanded: boolean, onToggle: () => void`
  - Collapsed row: name (bold), type badge (color-coded), level badge, rarity dot, bulk, price in gp, traits pills (first 3 + overflow count)
  - Expanded: full description (rendered via `stripHtmlInline`), type-specific stats:
    - Weapon: Damage formula, Category, Group
    - Armor: AC +N, Dex cap, Check penalty, Speed penalty, Str req
    - Consumable: Uses (N/max), Category
    - Equipment/Kit/Treasure/Backpack: description only
  - Type badge colors: weapon=red, armor=blue, consumable=green, equipment=gray, treasure=yellow, shield=orange, kit=purple, book=slate

- `src/entities/item/index.ts` — export ItemCard, ItemRow, ITEM_TYPE_LABELS, ItemTypeFilter

## Notes

- `stripHtmlInline` already defined in CreatureStatBlock.tsx — copy same local impl into ItemCard (no shared util to avoid FSD violation)
- Collapsed row height: ~40px, expandable via Collapsible (same pattern as SpellCard)
- No @-token resolution in ItemCard yet — raw description shown; Phase 24 adds full resolution
