# Plan 21-02: Full ItemsPage UI

**Phase:** 21 — Items Catalog Page
**Requirements:** ITMCAT-01, ITMCAT-02, ITMCAT-03, ITMCAT-04
**Status:** Not started

## Goal

Replace ItemsPage placeholder with a functional catalog — FTS5 search, type/level/rarity filters, scrollable list of ItemCard rows. Mirrors SpellsPage architecture.

## Changes

- `src/pages/items/ui/ItemsPage.tsx` — replace placeholder with:
  - Search input (200ms debounce, same pattern as SpellsPage)
  - Type filter pills: All / Weapon / Armor / Consumable / Equipment / Treasure / Shield / Kit (color-coded, toggleable)
  - Level range inputs: Min / Max (0–25)
  - Rarity filter: All / Common / Uncommon / Rare / Unique (pill toggles)
  - Scrollable results list using ItemCard with expand/collapse
  - Footer count: "N items" total / "N results" when filtered
  - Loading state + empty state ("No items found")
  - Calls `searchItems()` from shared/api on filter change

## Notes

- State: query (string), selectedType (ItemTypeFilter | null), levelMin (number | null), levelMax (number | null), selectedRarity (string | null), expandedId (string | null)
- useEffect on [query, selectedType, levelMin, levelMax, selectedRarity] → searchItems(...)
- Virtualization NOT needed at this stage — LIMIT 100 from API keeps DOM manageable
- Type pill colors match ItemCard badge colors defined in 21-01
